# Grok Research Workflow for This Project

## Use this order
1. Read 00_START_HERE.md
2. Read FILE_MANIFEST.md
3. Read the current front-end docs
4. Read the program and outline docs
5. Search the long manuals only as needed

## Research modes

### Mode A - Evidence gathering
Task: map the best current literature, guidelines, disagreements, and evidence gaps.
Use citations for every factual claim.
Prefer primary sources, guidelines, systematic reviews, meta-analyses, and large reviews.

### Mode B - Architecture critique
Task: identify conceptual holes, redundancies, false precision, tone mismatches, and weak front-end logic.
Be adversarial but concrete.

### Mode C - Paper drafting
Task: produce long-form papers that are dense, coherent, source-grounded, and stylistically strong.
Do not become vague, inflated, or pseudo-profound.

### Mode D - Rewrite / compression
Task: turn long materials into cleaner public-facing docs, onboarding pages, scripts, or form flows.

## Core constraints Grok should preserve
- realism over inspiration
- systems language over moral language
- function over sentimentality
- sequence over intensity
- small repeated actions over heroic bursts
- respect for low-energy execution
- respect for hope as effort -> outcome prediction
