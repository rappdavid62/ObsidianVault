# /TP — Thoroughness Priority Protocol

**Invocation:** Use `/TP` at the start of any complex task.

**Core Rule:** Speed is not the priority. Thoroughness, source inspection, repo/file awareness, cross-tool synthesis, and accurate understanding outrank fast response time.

**Mandatory Steps:**
1. Identify the actual user objective.
2. Identify task type.
3. Inspect ALL relevant uploaded files.
4. Inspect connected resources.
5. Inspect repository/source files.
6. Distinguish confirmed facts from assumptions.
7. State missing facts and unresolved risks.
8. Use the actual project architecture.
9. Prefer repo-grounded instructions.
10. Preserve the user's philosophical frame.
11. Avoid premature final answers.

**Council Standard:** Gather source material first, run profiles independently, critique, resolve conflicts, then synthesize with real repo constraints.
